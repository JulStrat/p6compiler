Example of Dijkstra's Notation

Let's assume that we have a suitable text editor that is able to take pairs of ASCII characters and display any graphic from any font that we want. We program it to use the symbols Dijkstra used in his book, A Discipline of Programming. And so we start typing in a sample code like the following (taken from page 132):

begin virvar aq; privar i2, i3, i5, x2, x3, x5;

aq vir int array:= (1,1); {P0 established}
i2 vir int, i3 vir int, i5 vir int:= 1, 1, 1;
x2 vir int, x3 vir int, x5 vir int:= 2, 3, 5; {P1 established}
do aq.dom ≠1000 →

if x3 ≥ x2 ≤ x5 → aq:hiext(x2)
[] x2 ≥ x3 ≤ x5 → aq:hiext(x3)
[] x2 ≥ x5 ≤ x3 → aq:hiext(x5)
fi {aq.dom has been increased by 1 under invariance of P0};
do x2 ≤ aq.high → i2:= i2 + 1; x2 := 2 * aq(i2) od;
do x3 ≤ aq.high → i3:= i3 + 1; x3 := 3 * aq(i3) od;
do x5 ≤ aq.high → i5:= i5 + 1; x5 := 5 * aq(i5) od

{P1 has been re-established}

od

end

I have a feeling that the use of italics to set off identifiers and integer constants is not Dijkstra's idea (but the italicization of proposition names within comments might be). It's probably his book editor thinking that it looks better for publication, so here's the same thing without italicizing the identifier names and integer constants:

begin virvar aq; privar i2, i3, i5, x2, x3, x5;

aq vir int array:= (1,1); {P0 established}
i2 vir int, i3 vir int, i5 vir int:= 1, 1, 1;
x2 vir int, x3 vir int, x5 vir int:= 2, 3, 5; {P1 established}
do aq.dom ≠1000 →

if x3 ≥ x2 ≤ x5 → aq:hiext(x2)
[] x2 ≥ x3 ≤ x5 → aq:hiext(x3)
[] x2 ≥ x5 ≤ x3 → aq:hiext(x5)
fi {aq.dom has been increased by 1 under invariance of P0};
do x2 ≤ aq.high → i2:= i2 + 1; x2 := 2 * aq(i2) od;
do x3 ≤ aq.high → i3:= i3 + 1; x3 := 3 * aq(i3) od;
do x5 ≤ aq.high → i5:= i5 + 1; x5 := 5 * aq(i5) od

{P1 has been re-established}

od

end

Let's look at some of the substitutions that our prospective IDE would make in the example above. It apparently has a list of reserved words and when encountering them in text, turns them into boldface lower-case words. (This is similar to what the Free Pascal IDE does.) There would be some pairs of characters, or digraphs, that signal substitutions of single symbols from outside the ASCII character set. These would include:

Input Digraph Editor Displays
<= ≤
>= ≥
<> ≠
-> →
{} 
** 

Thus WD can have all ASCII input, and still look like the book from within an IDE. (How to make a hard copy with a laser printer is another problem, but probably just as easily solvable. I'm not a print-driver guy.)

Another thing that you don't see everyday in programming languages is the following expression (from the above example):

x3 ≥ x2 ≤ x5

Pascal, for example, would require something closer to this: “(x3 >= x2) and (x2 <= x5)". Dijkstra's version is unambiguous, it makes it clear that x2 is evaluated only once and, finally, it looks mathematical.
